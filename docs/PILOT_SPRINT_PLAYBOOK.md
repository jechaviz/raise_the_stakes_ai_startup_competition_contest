# Pilot Sprint Playbook

Prepared: 2026-05-29

Goal: produce judge-ready validation for ContestOps AI without leaking private
founder, legal, customer, or portal data.

## Sprint Outcome

By the end of the sprint, collect:

- five target-user conversations;
- two written validations or LOIs;
- one live opportunity run selected by a user;
- one before/after metric set;
- one paid sprint, pilot-intent, or strong willingness-to-pay signal;
- one redacted proof page ready for RAISE.

## Operating Deadline

Use June 10, 2026 as the internal hard deadline because the VC4A listing shows
that close date. The RAISE FAQ page also shows July 1, 2026, so final portal
truth must be captured before submission.

## Pilot ICPs

| Segment | Pain to test | Best ask |
|---|---|---|
| Seed/pre-seed AI founder | Too many applications, not enough proof packaging | "Give me one opportunity you might skip and I will turn it into a submit-ready plan." |
| Accelerator applicant | Needs clean answers, team proof, deck/video, portal readiness | "Let us pressure-test your current application against the rubric." |
| Grant writer or proposal consultant | Manual intake, research, drafting, client follow-up | "Let us generate the evidence-gap report and first packet so you review, not coordinate." |
| Startup studio/operator | Repeated portfolio applications | "Let us run the same workflow across two portfolio companies or opportunities." |
| Accelerator/program manager | Receives incomplete or low-signal applications | "Let us test whether pre-submit guidance improves application completeness." |

## Offer Menu

| Offer | Timebox | Deliverables | Conversion ask |
|---|---:|---|---|
| Diagnostic | 30 minutes | Fit score, deadline risk, missing-proof list | Written feedback quote |
| Live Run | 90 minutes | Opportunity packet, evidence gaps, answer outline | Permission to cite redacted metrics |
| 48-Hour Sprint | 2 days | Submit-ready packet, asset checklist, payload, review pass | Paid pilot or LOI |
| Team Pilot | 1 week | Three opportunity packages and ops dashboard | Accelerator/studio subscription discussion |

## Qualification Filter

Run the pilot only if at least three are true:

- deadline within 45 days;
- clear upside: funding, customer, investor, partner, or program access;
- public opportunity page or shareable brief exists;
- applicant has enough product/team info for a credible first pass;
- user agrees to give structured feedback;
- user can approve a redacted metric or quote.

## Sprint Timeline

| Window | Action | Output |
|---|---|---|
| 0-4h | Build target list of 25 people across the ICPs | Names, segment, warm path, opportunity type |
| 4-12h | Send outreach and book async reviews | Five scheduled or async validations |
| 12-24h | Run diagnostics on two user-selected opportunities | Fit score, missing proof, estimated hours saved |
| 24-36h | Complete one live run end to end | Source, checklist, answer outline, evidence gaps |
| 36-48h | Ask for validation and willingness-to-pay | Two quotes/LOIs/pilot-intent messages |
| 48-60h | Package redacted proof | Metrics table, redacted quote, before/after |
| 60-72h | Update RAISE packet | Traction summary and proof readiness notes |

## Outreach Script

Subject: 30-minute application sprint test

Message:

I am validating ContestOps AI, an application-ops agent for founders. It turns a
startup competition, accelerator, grant, hackathon, or RFP into a submit-ready
workspace: rules, evidence gaps, draft answers, asset checklist, and form prep.

Could I run one opportunity you are considering and send back the result? I am
looking for blunt feedback on whether this would save time or improve your odds.

## Live Run Protocol

1. Capture the opportunity URL or brief.
2. Ask for permission boundaries: what can be public, private, or not stored.
3. Extract deadline, eligibility, rubric, fields, attachments, and risky claims.
4. Produce the first packet: checklist, evidence gaps, answer outline, asset
   list, and submission plan.
5. Review with the user and mark every claim as proven, private, missing, or
   remove.
6. Ask for validation, willingness-to-pay, and one redacted metric.

## Metrics

Track every pilot with:

- time from URL to first packet;
- number of required rules extracted;
- number of evidence gaps found;
- number of draft answers or field responses generated;
- number of portal fields mapped, if applicable;
- founder-reported hours saved;
- confidence before and after, 1-5;
- willingness-to-pay range;
- whether the user would use it on a real deadline.

## Acceptance Thresholds

| Signal | Green | Yellow | Red |
|---|---|---|---|
| User urgency | Has a real deadline and asks for next step | Interested but no active deadline | Abstract curiosity only |
| Workflow value | Says it changes go/no-go or submission quality | Says it saves drafting time | Says it is just nicer copy |
| Evidence value | Missing-proof list is useful | Some gaps useful | User already had all proof organized |
| Payment | Paid sprint, LOI, or budget owner intro | Would pay later | No willingness to pay |
| Repeatability | Same artifacts work across opportunities | Some customization needed | Bespoke consulting dominates |

## Validation Prompts

Ask after the run:

- What manual work did this remove?
- What output was most valuable: fit score, evidence gaps, answers, assets, or
  form prep?
- Would this have changed whether you applied?
- Would you pay for a 48-hour sprint before a real deadline?
- May we cite a redacted version of your feedback?

## Evidence Handling

Use public docs only for:

- segment;
- opportunity type;
- redacted outcome;
- aggregate metrics;
- approved quote or paraphrase.

Keep private:

- names unless explicitly approved;
- company legal data;
- funding data;
- customer proof;
- private portal screenshots;
- unsubmitted application contents;
- emails, calendars, and credentials.

## RAISE Packaging

Final traction line should be specific:

"In a 72-hour validation sprint, ContestOps AI ran [N] founder/application
workflows, found [N] evidence gaps, generated [N] submit-ready artifacts, and
received [N] written validations including [paid/LOI/pilot-intent signal]."

If the signal is weaker, say that clearly and use the sprint as proof of
learning velocity rather than overstating traction.
